1.2 РАСЧЕТНЫЕ ПАРАМЕТРЫ (Полная расшифровка)
────────────────────────────────────────────────────────────
📐 Формула 1: Размер кластера = 512 байт/сект × 8 сект/класт = 4096 байт
📐 Формула 2: Общий объем диска = 128895 сект × 512 байт = 65994240 байт (0.06 ГБ)
📐 Формула 3: Смещение $MFT = Кластер 5370 × 4096 байт = 0x14FA000 байт
              Абсолютный адрес $MFT = 0x10000 (Начало) + 0x14FA000 = 0x0150A000

====================================================================================================
                               ПУНКТ 1: МЕТКА ТОМА (ПОДРОБНЫЙ РАСЧЕТ)
====================================================================================================
Найти метку тома можно в системном файле $Volume (обычно это запись MFT #3).
📍 Читаем запись $Volume (MFT #3):
   - Абсолютный адрес MFT записи: 0x0150AC00
   - Внутри записи ищем атрибут 0x60 ($VOLUME_NAME).
   - Заголовок атрибута 0x60 найден по смещению: +0x100 от начала MFT-записи.
   - Смещение до самих данных имени (Offset to Data): +0x18 байт от заголовка атрибута.
   - Длина данных имени (Length of Name): 12 байт [HEX: 0xC].

Сырые байты имени (UTF-16LE): 46 00 53 00 48 00 44 00 48 00 42 00
📀 Итоговая декодированная метка тома: >> FSHDHB <<

====================================================================================================
                 ПУНКТ 3: СТРУКТУРА И АДРЕСАЦИЯ ОБЛАСТЕЙ ТОМА (РАСЧЕТЫ)
====================================================================================================
В файловой системе NTFS всё является файлами, поэтому фиксированных областей мало.
Ключевые области базируются на размере кластера и LCN (Logical Cluster Number).

1. Основной загрузочный сектор (Boot Sector / $Boot):
   - Описание: Содержит BPB (BIOS Parameter Block) и загрузочный код.
   - Абсолютное начальное смещение: 0x00010000
   - Стандартный размер в NTFS: 8192 байт (16 секторов) [HEX: 0x2000]

2. Главная таблица файлов ($MFT):
   - Описание: Реляционная база данных, содержащая записи обо всех файлах тома.
   - Из BPB получено значение стартового кластера LCN для $MFT: 5370 [HEX: 0x14FA]
   - Размер 1 кластера: 4096 байт [HEX: 0x1000]
   - Математика (Смещение от начала раздела): 5370 * 4096 = 21995520 байт [HEX: 0x14FA000]
   - Абсолютный адрес: 0x00010000 (Начало раздела) + 0x14FA000 = 0x0150A000

3. Резервная зона MFT (MFT Zone):
   - Описание: Зарезервированное пространство вокруг MFT (по умолчанию 12.5% диска) для предотвращения фрагментации.
   - Размер зоны: 8249280 байт (7.87 МБ)
   - Конец зоны: Абс. Смещение 0x01CE7FC0

4. Область данных (Data Area):
   - Описание: Начинается сразу после зарезервированных загрузочных секторов.
   - Кластеры адресуются с LCN = 0 (указывает на сам Boot Sector).
   - Всего секторов на диске: 128895 [HEX: 0x1F77F]
   - Всего кластеров для хранения: 16111 [HEX: 0x3EEF]
   - Все пользовательские файлы размещаются в свободных кластерах этой области через механизм Data Runs.
